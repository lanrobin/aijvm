# AIJVM

A minimal Java Virtual Machine built from scratch in C++20 — with **zero lines of hand-written code**. Every line of C++ was generated by AI (Claude / Gemini) from natural language requirements listed in the [docs/](docs/) directory.

## What is this?

AIJVM is a working JVM interpreter that can execute compiled Java `.class` files. It implements the core subsystems described in the *Java Virtual Machine Specification, SE 26 Edition*:

- **Class Loader** — Parses `.class` files and reads from OpenJDK `.jmod` archives (via libzip)
- **Runtime Data Areas** — PC register, JVM Stack, Frames (local variables + operand stack), Heap
- **Execution Engine** — Interpreter with all 200+ JVM bytecode opcodes (switch-case dispatch)
- **Memory Management** — Semi-space Copying Garbage Collector (Cheney's algorithm) with STW safepoints
- **Multithreading** — 1:1 threading model mapping Java threads to OS threads via `std::thread`
- **Native Interface** — Built-in native methods for `System.out.println`, `StringBuilder`, `Thread`, `Object.wait/notify`, etc.

## Prerequisites

### Linux (Ubuntu/Debian)

```bash
sudo apt update
sudo apt install build-essential git cmake gdb \
    libgtest-dev openjdk-25-jdk libzip-dev pkg-config
```

> spdlog is fetched automatically by CMake `FetchContent`. No manual install needed.

### Windows

- Visual Studio 2022+ with C++20 support
- CMake 3.20+
- vcpkg for dependencies: `vcpkg install gtest libzip spdlog`
- OpenJDK 25+ (for `javac` to compile test `.java` files and for `jmods/` path)

### macOS

```bash
brew install cmake googletest libzip pkg-config
brew install --cask temurin  # OpenJDK
```

## Build

```bash
mkdir -p build && cd build
cmake -DCMAKE_BUILD_TYPE=Debug ..
make -j$(nproc)
```

Or use the VS Code task:
```
Ctrl+Shift+B → cmake-build-debug
```

## Run

```bash
# Compile a Java program
cd tests/javaclasses
javac -XDstringConcat=inline HelloWorld.java

# Run it on AIJVM
cd ../../build
./aijvm --bj /usr/lib/jvm/java-25-openjdk-amd64/jmods \
        --cp ../tests/javaclasses \
        HelloWorld
```

Output:
```
Hello, World! The sum of 5 and 10 is 15
```

### With Garbage Collection

```bash
./aijvm --bj /usr/lib/jvm/java-25-openjdk-amd64/jmods \
        --cp ../tests/javaclasses \
        -Xmx10M GCTest
```

### CLI Options

| Option | Description |
|--------|-------------|
| `--bj <path>` | Path to OpenJDK `jmods/` directory (required) |
| `--cp <path>` | Path to user `.class` files directory (required) |
| `-Xmx<size>` | Maximum heap size (e.g., `-Xmx64M`, `-Xmx1G`). Enables semi-space GC. Default: NoOp GC. |

## Project Structure

```

  classfile/      Class file parser, JMOD reader
  classloader/    Dual-loader (bootstrap + application) with caching
  cli/            Command-line parser
  engine/         JVMEngine — top-level VM orchestration
  natives/        Native method bindings (Object, System, Thread, PrintStream, StringBuilder, Class)
  runtime/        Interpreter, Frame, Slot, JavaThread, Heap, GC, Safepoints, Opcodes
  utils/          Logger (spdlog wrapper), file reader
src/              Implementations (.cpp)
tests/            GoogleTest unit tests + Java test classes
docs/             AI prompt history — the only human-written files in this project
```

## How It Was Built

Each file in `docs/` is a natural language prompt given to Claude AI. The prompts follow a strict contract-driven workflow:

1. **Headers first** — AI proposes `.h` definitions; human reviews the API contract
2. **Implementation** — AI writes `.cpp` after approval
3. **Tests** — AI generates GoogleTest cases alongside every implementation
4. **Iterate** — Run, find errors, give error output back to AI, fix

The 18 prompt iterations built the JVM incrementally:

| # | Prompt | Subsystem |
|---|--------|-----------|
| 01 | CLI parsing | `command_line_parser` |
| 02 | Logging | `logger` (spdlog) |
| 03 | JMOD reading | `jmod_reader` (libzip) |
| 04 | Class file parsing | `class_file` (constant pool, attributes, methods) |
| 05 | Class loading | `class_loader` (bootstrap + application, caching) |
| 06 | Runtime data areas | `Slot`, `Frame`, `JavaThread` (locals, operand stack, PC) |
| 07 | Interpreter v1 | Basic opcodes: `iconst`, `iload`, `istore`, `iadd`, `return` |
| 08 | Wire into main | `JVMEngine` orchestration |
| 09 | Full interpreter | All 200+ opcodes, `Heap`, `NoOpGC` |
| 10 | Native methods | `NativeMethodRegistry`, `PrintStream.println`, `Object.<init>` |
| 11 | Class initialization | `<clinit>` state machine, `ensure_initialized` |
| 12 | Fix: StringBuilder | `StringBuilder` natives, `Class` natives, `-XDstringConcat=inline` |
| 13 | Multithreading | `JavaThread` + `std::thread`, 1:1 model |
| 14 | Fix: ProduceConsumer | `Thread.start/join`, `Object.wait/notifyAll`, `ACC_SYNCHRONIZED`, `monitorenter/monitorexit` with `std::recursive_mutex` + `condition_variable_any` |
| 15 | Garbage collection | Semi-space copying GC (Cheney's), `-Xmx`, `SafepointManager` |
| 16 | Fix: GCTest | `System.gc()` native, root scanning from Frame slots, forwarding pointer updates |
